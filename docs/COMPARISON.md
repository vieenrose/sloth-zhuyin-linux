# 懶音輸入法(Slothing) vs. Gboard 注音 vs. Boox 內建輸入法（Onyx kime / Rime）

Android 上三個可打注音的輸入法的誠實對照。事實均附來源；標「未驗證」者為
查無公開文件的項目。Slothing 的數字是本 repo 實測（BOOX Tab Mini C,
Android 11, arm64）。

## 總表

| | **Slothing** | **Gboard 注音** | **Boox 內建（com.onyx.kime, Rime）** |
|---|---|---|---|
| **解碼方式** | 從零訓練的 3.8M 雙向編碼器 LM,**免選字整句轉換**,逐字音對得上的文法約束 | ML 詞組/句型智慧選字（2018 起支援注音）[^gb1],詞組層級修正,非保證式整句轉換 | Rime `script_translator` 詞典＋語法插件的**連打整句**;基於詞典/n-gram,非訓練語料 LM [^rime1] |
| **免選字實測** | **74%**(172/230,微軟新注音免選字基準集,整句全對才計分) | 無公開可比數字(未驗證) | 無公開可比數字(未驗證) |
| **模型/詞庫大小** | **4.9 MB**(int8 ONNX,無損) | APK ~73–81 MB [^gb2];注音語言包大小未公開 | APK ~75 MB(全引擎);注音詞典源檔 ~1.8 MB [^rime2] |
| **離線** | **完全離線**(裝置端 ONNX Runtime,零網路權限需求) | 打字在裝置端處理;**聯邦學習回傳模型更新**與遙測(可關)[^gb3] | 完全離線;安全拆解實測**零按鍵外傳**、詞庫僅本機 [^kime1] |
| **開源** | **全開源**(引擎＋模型＋訓練管線,可完整重現) | 閉源 | Rime 引擎開源(kime 本身未公開源碼;Onyx 有拒供 GPL 源碼的紀錄 [^kime2]) |
| **候選列** | 即打即現:**句尾字候選**(點選即改)＋整句 n-best(點選上字);另有 ↓選字視窗(字/詞、‹›移焦、選字重評分) | 即打即現(詞組候選,展開格狀) | 即打即現(逐段候選,數字鍵選) |
| **聯想(次詞預測)** | 有,四前端(詞典 5178 字頭＋個人 bigram;行動點選、桌面 ⇧1-9,接龍上限 5) | 有(ML 次詞預測)[^gb1] | 核心無;需 librime-predict 外掛 |
| **選字後重評分** | **有**(字提示通道,整句圍繞你的選擇重解) | 無此概念(逐詞) | 無(逐段上屏) |
| **免聲調** | 有(模型靠語境消歧,70%) | 有(省略聲調=全聲調比對)[^gb4] | 有(schema 支援省聲調)[^rime3] |
| **打錯自動修正** | 有(模型評分的編輯距離 1 修復) | 有(ML 自動修正) | 無(需正確音節) |
| **個人化學習** | 本機 learn.tsv(校準 logit 加分) | 裝置端學習＋聯邦學習上傳 | 本機 Rime 使用者詞庫 |
| **e-ink 最佳化** | **有**(零動畫、平面高對比、固定候選列不位移) | 無(按鍵彈跳動畫等) | 由 Onyx 出品,e-ink 取向(細節未公開) |
| **注音在 Boox 上** | 裝了就有 | 官方台灣代理商的建議方案 [^kime3] | **國際版韌體不含注音**,依機型/地區而異 [^kime3] |

## 三句話總結

- **Gboard**:成熟、詞組級智慧選字,但閉源、帶遙測,且注音品質長年被視為
  遜於已下架的 Google 注音輸入法(坊間留有「裝回 APK」教學)[^gb5]。
- **Boox 內建(Rime)**:真正離線、零外傳,但解碼是詞典＋n-gram 的連打,
  沒有語料訓練的語言模型,同音字消歧靠詞頻;國際版韌體甚至不含注音。
- **Slothing**:唯一一個**訓練出來的 LM 做免選字整句轉換**、可測量
  (74% vs 微軟新注音答案集)、完全離線、全開源、4.9 MB,且與桌面
  fcitx5/IBus/網頁版共用同一個核心與模型。

## 方法備註

免選字 74% 的量法:`eval/reference_mspy.tsv` 230 句(微軟新注音/自然輸入法
能免選字正確轉換的句子=準確率天花板),整句逐字全對才計 1。裝置端結果與
桌面原模型逐句一致率 99%(僅 1 句浮點平手翻面)。Gboard/Rime 無法以同法
自動測量(閉源 UI/逐段上屏),故不給未驗證的數字。
